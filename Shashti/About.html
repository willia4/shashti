Shashti
The Humane Password Generator
By James Williams
Shashti replicates the Diceware password generation system by Arnold G. Reinhold in a somewhat more efficient manner than rolling dice over and over.
If you read the Diceware page, you'll see that the author explicitly recommends against using a computer to generate passwords. This application mitigates his concerns somewhat by using a system-provided pseudo-random number generator intended for cryptographic services.
It avoids skewing towards powers-of-two by "rolling dice" in the same manner as you would do if you actually had dice: it repeatedly requests a number from a uniform pseudo-random distribution of digits between 1 and 6 (inclusive) and chooses words from the Diceware list in exactly the same way as proscribed to be done manually.
This application also offers an alternative to the already extremely-secure pseudo-random number generator: it is able to fetch truly random numbers from RANDOM.ORG. RANDOM.ORG uses physical sources of noises to seed its number pool and is scientifically guranteed to be random to the extent that we understand physics.
However, based on the strength of the pseudo-random number generator, the I believe that this is unnecessary.
A far greater security risk is that this application may "phone home" and send your passwords to me. This is mitigated because the source code for this application should be freely available and you can see that it does no such thing.
For your own peace of mind, I also recommend that you generate multiple passwords at a time and then pick one from the list. In the worst case, a malicious attacker with access to your generated passwords will still need to try all of them.
